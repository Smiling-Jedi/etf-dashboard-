## 一、背景与目标

### 1.1 策略背景
- **策略来源**: 邢大ETF三因子轮动策略（原文宣称7年13-17倍收益）
- **实盘资金**: 10万人民币
- **核心问题**: 验证不同执行时点和阈值规则下的回测效果，确定最优实盘方案

### 1.2 三因子模型
| 因子 | 权重 | 计算方式 |
|------|------|----------|
| 乖离动量(Bias) | 30% | 收盘价/20日均线，25天线性回归斜率×10000 |
| 斜率动量(Slope) | 30% | 20天价格线性回归斜率×R²×10000 |
| 效率动量(Efficiency) | 40% | 25天价格动量×效率比(方向/波动) |

**排名规则**: 4只ETF三因子Z-Score标准化后加权求和，按总分排名

### 1.3 验证目标
1. 对比6种执行方式（不同时点组合）的回测效果
2. 验证1.5倍阈值规则的有效性
3. 解决"收盘后排名vs收盘价执行"的时间悖论
4. 确定实盘可执行的最优策略

## 二、关键Bug发现与修复

### 2.1 发现的数据泄露Bug
在初期回测中发现：**6-variations版本**与**17x版本**跑出不同结果

| 版本 | 净值 | 问题 |
|------|------|------|
| 17x版本 | 17.39x | 效率动量使用`df_hist`（正确） |
| 6-variations原版 | 11.89x | 效率动量使用`df`（数据泄露） |

**Bug原因**:
```python
# 6-variations原版（有Bug）
df_recent = df.iloc[-MOMENTUM_DAY:].copy()  # ❌ 用了完整数据df

# 修复后
df_recent = df_hist.iloc[-MOMENTUM_DAY:].copy()  # ✅ 用过滤后的df_hist
```

**修复后收益**: 从11.89x提升至**17.39x**

### 2.2 阈值判断Bug
`select_with_threshold()`返回`(target, should_trade)`元组，但`run_strategy()`只解包了一个值，导致阈值判断被绕过。

**修复**:
```python
# 修复前
target = select_func(factors, holding)

# 修复后
target, should_trade = select_func(factors, holding)
if should_trade and target != holding:
```

#### 1. 1.5倍阈值有效
- 带阈值版本交易次数更少（99次 vs 109次）

## 四、策略3收盘版详细说明

### 4.1 策略规则

| 项目 | 说明 |
|------|------|
| **排名计算** | 每周五收盘后，基于周五收盘价计算三因子并排名 |
| **调仓判断** | 带1.5倍阈值：新第1名得分>当前持仓得分×1.5才调仓 |
| **卖出执行** | 下周一 14:50-15:00，以收盘价卖出旧持仓 |
| **买入执行** | 下周一 14:50-15:00，以收盘价买入新排名第1 |
| **买入标的** | 周五排名得分最高的ETF（第1名） |

### 4.2 1.5倍阈值逻辑详解

```
场景：当前持仓得分0.6，新第1名得分0.9

判断流程：
1. 当前持仓>0分 → 进入正分判断分支
2. 检查：新第1名(0.9) > 当前(0.6)×1.5=0.9？
3. 0.9 = 0.9，不满足"大于" → 不调仓

场景：新第1名得分0.91
判断：0.91 > 0.9 → 满足条件，调仓到第1名
```

**核心思想**：只有当新第1名明显强于当前持仓（强1.5倍以上）时才换，避免频繁交易。
